You can use cli for more than just error messages! Can be helpful for providing semantically-formatted and nicely-presented info messages to package users.

check_string <- function(x) {
  cs <- "check_string"
  error_text <- "{.fn {cs}}: argument {.arg x} is not a valid string"
  if (rlang::is_string(x)) {
    x
  } else {
    cli::cli_abort(c(error_text, "x" = "You supplied {.code {x}}"))
  }
}

Nested checking functions ü™Ü

This might all seem a bit excessive, but when I‚Äôm building a package I will usually have little helper functions within bigger helper functions which in turn support the main functions that the user is going to interact with.

check_string <- function(x) {
  error_text <- c(
    "{.fn check_string}: argument {.arg x} is not a valid string",
    "x" = "You supplied {.code {x}}"
  )
  if (rlang::is_string(x)) {
    x
  } else {
    cli::cli_abort(error_text, call = rlang::caller_call())
  }
}

my_lower <- \(x) tolower(check_string(x))
